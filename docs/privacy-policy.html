Privacy Policy

Last updated: May 12, 2026

Overview

Calendar Fetch is a personal automation tool that reads Google Calendar events and sends a morning summary to a private Slack workspace. This policy explains how data is handled.

Data Collected

The application accesses the following data through the Google Calendar API:

  • Calendar event titles
  • Event start and end times
  • Event locations (if provided)
  • Event descriptions (if provided)

How Data Is Used

Calendar data is used solely to generate a daily morning summary message that is posted to a private Slack channel. The data is processed in memory at runtime and is not stored, logged, shared, or transmitted to any third party beyond the designated private Slack channel.

Data Retention

No calendar data is persisted. All event information is discarded immediately after the summary message is sent to Slack. No database, file, or external service retains the data.

Authentication Credentials

OAuth tokens used to access Google Calendar are stored locally on the machine running this tool and are never shared with any third party. These credentials are excluded from version control via .gitignore.

Third-Party Services

This tool interacts with the following services:

  • Google Calendar API — to read calendar events (read-only access)
  • Slack Incoming Webhooks — to deliver the summary to a private channel
  • Ollama — a locally-running language model for generating the summary text; no data leaves the local machine through this step

Contact

This is a personal project. If you have questions, please open an issue on the GitHub repository.
